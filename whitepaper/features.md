# Hunter · 功能详细说明

> 全能逗猫机器人 · 产品功能白皮书 · 2025

---

## 产品功能概览

全能逗猫机器人融合计算机视觉、边缘 AI 推理、机电控制与云端服务，面向现代宠物主人提供 7×24 小时的智能陪伴解决方案。本章节对三大功能模块（感知、逗猫、照料）下属的 11 项核心功能逐一进行 5W1H 详细阐述。

| 模块 | 功能项 | 核心价值 |
|------|--------|----------|
| 一、感知 | 视觉追踪 | 实时锁定猫咪位置，底盘自动跟随 |
| | 叫声识别 | 解析情绪状态，触发差异化响应 |
| | 活跃度感知 | 判断精力水平，自动切换逗猫强度 |
| 二、逗猫 | 逗猫棒挥舞 | 机械臂驱动，动作随机 + 学习偏好 |
| | 激光点追逐 | 激光联动底盘，全方位路径规划 |
| | 零食投喂奖励 | 正向强化，提升猫咪参与动力 |
| 三、照料 | 饮水监测 | 传感器追踪饮水，异常及时预警 |
| | 表情包生成器 | 自动捕捉最萌瞬间，一键分享 |
| | 猫咪每日日报 | 健康数据汇总，推送至主人手机 |
| | 猫咪个性偏好 | 长期学习，构建专属行为模型 |
| | 远程 APP 控制 | 出门在外随时查看与操控 |

## 模块一：感知

### 1.1 视觉追踪

摄像头模块持续捕捉画面，通过轻量级目标检测模型实时定位猫咪，驱动底盘和机械臂跟随调整，确保逗猫工具始终处于最佳交互距离。

| 维度 | 说明 |
|------|------|
| **WHO** | 用户家中独处的宠物猫及其主人（通过日志回看） |
| **WHAT** | 基于摄像头的实时猫咪位置检测与底盘自动跟随 |
| **WHEN** | 猫咪在视野内移动时持续触发，30 fps 实时推理 |
| **WHERE** | 室内任意区域：客厅、卧室、走廊，覆盖正常活动范围 |
| **WHY** | 解决逗猫工具固定位置导致猫咪失去兴趣的痛点；让机器人主动跟随，延长互动时长 |
| **HOW** | IMX219 摄像头 + YOLOv8-nano 模型在 Jetson Nano 上以 25 fps 推理；检测到猫咪质心后输出 (x, y) 坐标，PID 控制器驱动麦克纳姆轮底盘跟随，机械臂同步调整仰角 |

---

### 1.2 叫声识别

麦克风阵列持续采集环境声音，经过降噪预处理后送入声纹分类模型，区分饥饿、撒娇、警戒、满足等情绪类别，触发相应响应策略。

| 维度 | 说明 |
|------|------|
| **WHO** | 猫咪（声音发出方）与主人（接收提醒方） |
| **WHAT** | 猫咪叫声实时分类，区分至少 4 种情绪状态并触发差异化动作 |
| **WHEN** | 猫咪发出叫声时即时触发，静默期自动进入低功耗监听模式 |
| **WHERE** | 摄像头覆盖的室内区域；麦克风阵列收音范围约 5 米 |
| **WHY** | 让机器人真正「听懂」猫咪需求——饥饿叫声触发投喂提醒，警戒叫声让机器人后退停止，避免强迫互动造成应激 |
| **HOW** | 双麦克风阵列做波束成形降噪；MFCC 特征提取 + 轻量 CNN 分类器（准确率目标 ≥ 85%）；4 类输出：饥饿 → 推送提醒、撒娇 → 触发温和互动、警戒 → 停止并后退、满足 → 记录进日报 |

---

### 1.3 活跃度感知

融合视觉运动量估计与环境时间信息，实时评估猫咪当前精力水平，自动在「疯狂模式」与「轻柔模式」间切换，避免过度刺激或无效逗弄。

| 维度 | 说明 |
|------|------|
| **WHO** | 各年龄段、各性格的猫咪（幼猫 / 成猫 / 老猫需求差异显著） |
| **WHAT** | 基于光流运动量 + 时间段的实时活跃度评级（高 / 中 / 低），驱动逗猫强度自动切换 |
| **WHEN** | 每 10 秒更新一次评级；活跃度突变时（如从静止突然奔跑）即时响应 |
| **WHERE** | 室内全覆盖；结合时间段（清晨 / 黄昏为猫咪天然活跃期）做加权判断 |
| **WHY** | 高活跃期配合高强度逗弄可最大化运动量；低活跃 / 睡眠期自动暂停，避免打扰，延长机器人续航 |
| **HOW** | Lucas-Kanade 光流法计算全帧平均运动向量幅度；结合时间段权重得出活跃度分（0–100）；阈值 > 60 触发疯狂模式，< 20 触发暂停或轻柔模式；历史数据存入本地数据库用于个性化校准 |

## 模块二：逗猫

### 2.1 逗猫棒挥舞

机械臂末端夹持逗猫棒，结合随机运动生成算法与从历史交互数据中提炼的猫咪偏好模型，动态生成兼具不可预测性与针对性的运动轨迹，持续激发猫咪的捕猎本能。

| 维度 | 说明 |
|------|------|
| **WHO** | 具有追逐、扑抓本能的家猫（尤其适合室内缺乏运动刺激的猫咪） |
| **WHAT** | 机械臂驱动逗猫棒做三维随机运动，动作频率和幅度根据活跃度评级实时调节 |
| **WHEN** | 由活跃度感知模块触发；单次逗玩时长默认 15 分钟，可 APP 调整；疯狂模式下加速，轻柔模式下放缓 |
| **WHERE** | 机器人跟随视觉追踪定位于猫咪附近 0.3–0.8 米处；逗猫棒运动范围覆盖猫咪前方 180° 扇形区域 |
| **WHY** | 满足猫咪天然捕猎欲望，释放多余精力，减少因无聊引发的破坏行为（抓家具、过度梳毛等） |
| **HOW** | 4-DOF 机械臂由舵机驱动；运动轨迹由贝塞尔曲线参数化生成，随机扰动幅度 ±15°；偏好模型提取「猫咪扑抓成功率最高的速度区间」并加权采样，使每 3 次动作中约有 1 次可被猫咪「抓到」，维持成就感 |

---

### 2.2 激光点追逐

激光模块与底盘联动，在地面投射移动的红色光点，路径由算法生成以避免死角与重复，全方位激发猫咪的追逐欲望。

| 维度 | 说明 |
|------|------|
| **WHO** | 对光点敏感、具有追逐本能的猫咪（多数家猫均适用） |
| **WHAT** | 激光点在地面做不可预测路径游走，底盘同步移动确保照射角度，避免光点消失在家具阴影或墙角 |
| **WHEN** | 可独立运行，也可与逗猫棒模式交替切换；单次激光游走建议不超过 10 分钟，防止猫咪因无法「抓住」而产生焦虑 |
| **WHERE** | 室内地面全覆盖区域；激光仰角 < 15° 确保安全，不直射猫眼 |
| **WHY** | 激光游走对场地要求极低，适合小户型；提供高强度有氧运动，有效消耗精力；成本远低于物理玩具且无耗损 |
| **HOW** | 650nm 5mW 红色激光模块（Class II，安全级别）；搭载二轴云台控制投射角度；路径规划使用随机游走 + 禁忌搜索算法避免重复；检测到猫咪停止追逐超过 30 秒自动暂停，防止无效输出 |

---

### 2.3 零食投喂奖励

机械臂末端配备零食弹射模块，在猫咪完成一次有效追逐或扑抓后自动触发投喂，利用操作性条件反射原理强化互动行为，提升猫咪对机器人的配合度与信任感。

| 维度 | 说明 |
|------|------|
| **WHO** | 所有家猫；对食物动机强的猫咪效果尤为显著 |
| **WHAT** | 检测到猫咪「扑抓成功」后 1 秒内弹出零食颗粒，投喂量可精确控制（每次 1–3 粒） |
| **WHEN** | 在逗猫棒或激光追逐会话结束时、或猫咪完成高难度动作（跳跃 > 30cm）后触发 |
| **WHERE** | 零食落点控制在猫咪前方约 20 cm，确保其能顺利找到并取食 |
| **WHY** | 正向强化机制提升猫咪主动参与意愿；配合日报中的卡路里追踪功能，杜绝过度喂食；同时增强猫咪对机器人的正面联结 |
| **HOW** | 弹射机构由步进电机 + 螺旋推进器实现定量出料；储料仓容量约 50 粒，可拆卸清洗；每日投喂上限可在 APP 中设置，超限自动锁定；零食重量传感器检测余量并在 APP 端提示补充 |

## 模块三：照料

### 3.1 饮水监测

通过部署在水碗旁的非接触式液位传感器，持续追踪猫咪饮水频率与饮水量，结合天气温度数据生成每日推荐饮水量，异常时向主人推送预警。

| 维度 | 说明 |
|------|------|
| **WHO** | 主人（接收数据与预警）；猫咪（被监测对象，泌尿系统问题高发） |
| **WHAT** | 实时记录每次饮水的时间与估算饮水量；若 12 小时内饮水次数为零，触发 APP 推送；机器人同时主动驶向猫咪并发出轻声提示引导其饮水 |
| **WHEN** | 7×24 小时持续监测；数据每小时聚合一次存入本地数据库 |
| **WHERE** | 水碗固定放置位置；传感器夹装于碗沿，无需改装水碗 |
| **WHY** | 猫咪因饮水不足引发的泌尿系统疾病（FIC、尿路结石）是头号健康风险之一；早期发现饮水异常可显著降低就医成本与猫咪痛苦 |
| **HOW** | 超声波液位传感器（±1mm 精度）每 30 秒采样一次；液位下降幅度 × 水碗截面积换算饮水量（ml）；数据经 MQTT 协议推送至云端聚合；异常判定阈值可依猫咪体重在 APP 中自定义 |

---

### 3.2 表情包生成器

摄像头持续以滑动窗口缓冲最近 2 秒画面，当检测到猫咪姿态突变（扑抓、跳跃、瞪眼等高价值瞬间）时，从缓冲中回溯提取萌度最高帧，经视觉语言模型自动生成配文，合成可一键分享的表情包图片。

| 维度 | 说明 |
|------|------|
| **WHO** | 猫咪主人（内容消费与分享者）；社交媒体受众（二次传播） |
| **WHAT** | 全自动捕捉猫咪精彩瞬间，生成带配文的表情包图片，推送至 APP 相册 |
| **WHEN** | 触发条件：光流运动量 > 阈值 AND 萌度评分 > 0.7；每日上限 20 张防止刷屏；主人也可在 APP 手动触发「随机抓拍」 |
| **WHERE** | 机器人摄像头覆盖的全部室内区域 |
| **WHY** | 将日常陪伴转化为可传播的情感内容，强化主人与宠物的情感联结；同时作为产品自然传播渠道，每张分享图均含品牌水印（可关闭） |
| **HOW** | 30fps 环形缓冲（60 帧 / 2 秒）；YOLOv8-nano 检测猫咪 BBox；Lucas-Kanade 光流估算运动量；自研萌度评分模型（输入：正脸置信度 + 双眼占比 + 嘴部开合）；触发后取评分最高帧发送给视觉语言模型（LLaVA / GPT-4o Vision）生成 3 种风格配文（严肃体 / 自恋体 / 茫然体）；Pillow 合成最终图片 |

---

### 3.3 猫咪每日日报

每天定时汇总当日所有传感器与 AI 分析数据，自动生成一份结构化的「猫咪健康日报」，以卡片形式推送至主人手机 APP，让主人即使不在家也能掌握爱猫全天状态。

| 维度 | 说明 |
|------|------|
| **WHO** | 猫咪主人，尤其是白天长时间不在家的上班族 |
| **WHAT** | 每日一份包含运动量、互动时长、饮水次数、情绪分布、表情包精选的综合健康报告 |
| **WHEN** | 每天 20:00 定时生成推送；若当日出现异常事件（饮水为零、持续警戒叫声等）则即时插播预警 |
| **WHERE** | APP 通知 + 应用内卡片；历史数据可在 APP 中按周 / 月查看趋势图 |
| **WHY** | 将碎片化传感器数据转化为主人可读的健康洞察；早期发现行为异常（如活跃度骤降、持续嗜睡）辅助判断是否需要就医 |
| **HOW** | 本地数据库（SQLite）按日聚合：运动量（光流累计）、互动时长（逗猫模块日志）、饮水次数、情绪分类统计、最萌表情包 TOP 3；LLM 模板填充生成自然语言摘要；Firebase Cloud Messaging 推送至 iOS / Android |

---

### 3.4 猫咪个性偏好

通过对猫咪长期行为数据的持续学习，系统逐步构建该猫咪的专属偏好模型，覆盖玩耍风格、作息规律、食物偏好等维度，使逗猫策略随时间演进，越用越懂你的猫。

| 维度 | 说明 |
|------|------|
| **WHO** | 每只猫咪（偏好高度个体化）；主人（通过 APP 查看与手动标注） |
| **WHAT** | 持续学习并更新猫咪偏好画像：最爱的逗猫棒速度区间、活跃高峰时间段、对激光 vs 物理玩具的偏好比、最爱零食口味（需主人录入） |
| **WHEN** | 每次互动会话结束后增量更新模型；每周生成一份「偏好洞察」推送给主人；累计数据 ≥ 7 天后模型预测精度显著提升 |
| **WHERE** | 本地设备存储（隐私优先）；可选择加密上传至云端做跨设备同步 |
| **WHY** | 通用逗猫策略对不同猫咪效果差异巨大；个性化模型可将「猫咪主动参与时长」提升 30–50%（目标指标）；同时为主人提供了解爱猫的新视角 |
| **HOW** | 特征工程：从每次会话提取扑抓成功率、持续参与时长、情绪状态分布；在线学习算法（轻量级 Bandit 模型）根据反馈信号更新各动作的期望奖励；偏好向量存储于本地 SQLite；主人可在 APP 中查看雷达图并手动修正异常标注 |

---

### 3.5 远程 APP 控制

通过专属手机 APP，主人可随时随地实时查看猫咪画面、手动触发或暂停逗猫程序、调整参数设置，以及接收各类通知，无论身处何处都能掌握主动权。

| 维度 | 说明 |
|------|------|
| **WHO** | 外出上班、出差或旅行的猫咪主人 |
| **WHAT** | 完整的远程监控与控制能力：实时视频流、手动逗猫、参数调整、通知管理、历史回放 |
| **WHEN** | 7×24 小时可用；延迟目标 < 500ms（同城 4G/5G 网络下） |
| **WHERE** | iOS / Android APP；家庭 Wi-Fi + 云端中继（处理 NAT 穿透） |
| **WHY** | 消除主人「猫咪在家独处」的分离焦虑；在紧急情况（猫咪异常叫声、饮水为零预警）下可立即远程介入；同时允许主人参与逗猫乐趣，而非完全委托给机器 |
| **HOW** | 本地端：RTSP 视频流经 WebRTC 编码；控制指令通过 MQTT over TLS 加密传输；云端：轻量级中继服务器处理 P2P 打洞（失败则回退至中继）；APP 端：React Native 跨平台开发；功能模块：实时画面、快捷逗猫按钮、参数面板（逗猫时长 / 零食上限 / 激光模式）、通知中心、7 日回放功能 |

## 功能整合：协同工作示意

上述 11 项功能并非孤立运行，而是围绕「感知 → 决策 → 执行 → 反馈」闭环紧密协作：

```
感知层输入
视觉追踪 + 叫声识别 + 活跃度感知
持续提供猫咪状态数据
         ↓
决策层
个性偏好模型
综合感知数据，选择当前最优逗猫策略
         ↓
执行层
逗猫棒挥舞 / 激光点追逐 / 零食投喂
按策略协调执行
         ↓
反馈层
表情包生成器 + 每日日报 + 饮水监测
将执行结果结构化，回流至偏好模型与主人
         ↓
人机接口
远程 APP 控制
让主人随时介入，提供人工标注辅助模型迭代
```

这一闭环架构使机器人从「被动执行工具」演进为「主动学习伙伴」，差异化竞争壁垒随使用时长持续加深。
