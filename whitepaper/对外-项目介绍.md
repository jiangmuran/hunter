# Hunter · 项目介绍

> 面向评委、投资人、合作方 · V1.0 · 2026年5月

---

## 项目一句话介绍

**基于具身 AI 的消费级猫咪陪伴代理——用生成式动作和非语言感知，填满主人不在的那九个小时。**

Slogan：**「你不在的时候，它在。」**

---

## 一、我们在解决什么问题

中国有超过 1 亿只宠物猫，其中大量生活在一线城市公寓里，主人每天工作 9 小时以上。

猫天生是猎手。它的神经系统为追逐、伏击、博弈而设计，但现代公寓给不了它猎物。长期缺乏有效刺激，会导致肌肉萎缩、肥胖、抑郁、焦虑性破坏行为——这是室内猫群体中正在蔓延的隐性健康危机。

市面上的解决方案逻辑都一样：**给猫一个规律的刺激**。自动逗猫棒按固定轨迹转，激光随机乱跑。猫的智商足以在三天内识破任何规律。三天之后，它再也不看一眼。

传统玩具给的是"刺激"。猫真正需要的是**博弈**——一个会逃跑、会反击、永远无法被完全预测的对手。

**Hunter 就是这个对手。**

## 二、核心卖点（对齐评审维度）

### 卖点 1：完整的生物学闭环——现有产品无一做到

Hunter 构建了完整的**狩猎 → 捕获 → 进食**闭环：机械臂模拟猎物动态，猫扑中时底盘投出零食作为奖励。这不是噱头——这是模拟真实捕食行为的唯一方式，也是让猫持续参与互动而不是三天放弃的关键。

市面上没有任何产品做到了这一点。

→ 对应评审维度：**创新性**

### 卖点 2：AI 必须主动制造意外——惊喜熵引擎

所有 AI 产品都在优化：ChatGPT 追求"最准确"，推荐算法追求"最点击"。**但优化是猫无聊的根源。**

Hunter 引入全新指标：**"惊喜熵"（Surprise Entropy）**——实时衡量当前动作对这只猫的不可预测程度。算法目标不是"让猫扑得更准"，而是"让猫始终保持 70% 把握 + 30% 不确定"。

这是 Csikszentmihalyi 心流理论在物理交互上的工程实现，把游戏设计、即兴喜剧原理与具身 AI 融合——相关文献几乎空白。

Hunter 会故意失败（让猫扑中）、故意打破规律（在猫预测下一步前反转）、故意"笨拙"（模拟受伤猎物）。当它发现某个套路猫已熟练，立刻主动弃用。

→ 对应评审维度：**技术难度 + 创新性**

### 卖点 3：第一个为"不会说话的存在"设计的 AI

现有所有 AI 产品——ChatGPT、Siri、Copilot——都依赖**语言**作为接口。Hunter 读的是尾巴姿态、耳朵角度、瞳孔大小、全身体态。

这是 Ambient Computing 的极致：AI 在没有任何主动指令的情况下读懂主体。

它的英文定义：**"AI for the Inarticulate."**

今天做猫，明天技术栈可平移到失能老人、低龄婴儿、特殊需求儿童——伦理光环加成，市场无限大。

→ 对应评审维度：**商业潜力（长期赛道想象力）**

### 卖点 4：每台 Hunter 最终都不一样——真正的 AI 原生硬件

传统硬件：出厂是什么样，用一年还是什么样。Hunter 出厂时相同，但和"它的那只猫"持续共同进化，积累专属的人格记忆和行为偏好档案。

六个月后，张三家的 Hunter 学会了这只猫不喜欢被快速逼近，李四家的 Hunter 学的是另一套。**两台设备的"灵魂"已经不一样了。**

商业意义：用户不能换设备——换了就失去六个月的人格化记忆。这是比订阅更深的 SaaS 锁定，时间站在我们这边。

→ 对应评审维度：**商业潜力（护城河）**

### 卖点 5：AI 日记和记忆盒子——情感钩子

每晚推送的不是数据图表，是 LLM 写的文字：

> "今天 Mochi 在阳光下睡了两小时，醒来后试图捕捉那只羽毛玩具 23 次，差两次几乎得手，失手后假装根本不在乎。"

Hunter 在猫的整个生命中持续记录每一个互动瞬间。猫离世后自动整理"它的一生"。营销时只写一句话：**"Hunter 会记得它生命的每一天。"**

→ 对应评审维度：**现场讲述效果（情感穿透力）**

## 三、产品能力全览

### 核心流程

```
摄像头持续扫描房间
         ↓
   视觉追踪 + 叫声识别
（姿态 / 情绪 / 活跃度判断）
         ↓
   状态机决策：是否启动互动
         ↓
   生成式动作模拟猎物
（惊喜熵实时调节，永不重复）
         ↓
   猫扑中 → 底盘投食
（狩猎 → 捕获 → 进食闭环）
         ↓
   腕摄全程录制高光瞬间
         ↓
   AI 自动剪辑 + 生成表情包
         ↓
   每晚推送 AI 日记 + 高光视频
```

### 功能模块

| 模块 | 说明 | 优先级 |
|------|------|--------|
| 视觉追踪 | YOLOv8 实时定位猫的位置和状态 | P0 |
| 叫声识别 | 麦克风分类饥饿/撒娇/警戒/玩耍 | P0 |
| 生成式逗猫 | 惊喜熵驱动，机械臂逗猫棒/激光切换 | P0 |
| 零食闭环 | 扑中时底盘自动投食 | P0 |
| 高光捕捉 | 腕摄识别关键帧（扑跳/翻滚/回头） | P1 |
| 表情包生成 | 自动裁剪 + 文字叠加 + 存入相册 | P1 ✅ |
| AI 日记 | LLM 每日生成文字日记推送主人 | P1 |
| 饮水监测 | 摄像头分析饮水频次异常预警 | P1 |
| 远程接管 | App 第一视角实时操控 | P2 |
| 记忆盒子 | 生命全程记录，自动整理回忆录 | P2 |
| 个性档案 | 长期学习专属偏好，随猫进化 | P2 |

## 四、目标用户

**V1 唯一目标用户：**

- 25–35 岁，一线 / 新一线城市
- 独居或同居无孩
- 1 只 1–8 岁活力健康猫
- 工作日 9 小时以上不在家
- 能接受 ¥1500–2500 的智能家居单品

**明确不服务（边界即护城河）：**

多猫家庭（V2 再做）、老年猫（安全要求不同）、病猫/术后猫（医疗器械范畴）、狗（运动尺度差异太大，做不好两边都伤）、老人独居（产品交互复杂度不适配）。

评委若问"你们也支持狗吗"——正确答案是"V1 不做，因为狗的运动尺度和猫差异太大，做不好两边都伤"。**这种"敢说不"比"什么都行"得分高。**

## 七、路演策略

### EXPO 阶段（30 秒抓住评委）

> "你有没有想过，你每天上班的九个小时，你的猫在做什么？它是猎手，但家里没有猎物。我们做了一个会逃跑的机器人。来，我演示给你看——把手伸过去抓它。"

**关键动作：让评委现场体验躲闪。** 机械臂的瞬间反应是任何 PPT 都替代不了的视觉冲击。

### 决赛阶段（路演结构）

| 时间 | 内容 | 要点 |
|------|------|------|
| 0–1 min | **痛点故事** | "三天之后，它再也不看一眼"——打出反直觉的核心矛盾 |
| 1–3 min | **演示** | 现场展示：评委伸手 → 机械臂躲开；然后展示 AI 日记样例 |
| 3–5 min | **技术深度** | 惊喜熵引擎、非语言感知、生物学闭环 |
| 5–6 min | **商业逻辑** | "用户不能换设备"——时间站在我们这边 |
| 6–7 min | **更大的图景** | AI for the Inarticulate——从猫到老人到婴儿 |
| 7–8 min | **记忆盒子** | "Hunter 会记得它生命的每一天"——收尾，让全场安静 |

## 八、风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|---------|
| YOLO 检测延迟影响体验 | 中 | 赛前测试好推理速度，用 YOLOv8 nano；备用：纯运动检测降级方案 |
| 机械臂在演示中异常 | 低 | 提前录制一段完整演示视频作为 fallback |
| 评委觉得"就是个逗猫机器人" | 中 | 重点展示惊喜熵和非语言感知——这是别的逗猫机器人做不到的 |
| 评委问"为什么不做狗" | 低 | 准备好"V1 不做，聚焦比宽泛更有力量"的回答 |
| 投食机构现场卡顿 | 低 | 演示中用手动投食替代，聚焦算法部分的展示 |
